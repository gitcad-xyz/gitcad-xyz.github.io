Three bets

Every incumbent CAD tool is a GUI with an API bolted on, a proprietary binary format, and a subscription wall around collaboration. gitcad inverts all three.

01 · Agent-first, MCP-native

The MCP tool surface is the primary interface — the Python API and the web UI are clients of the same handlers. Agents model by verifying, not hoping: every operation returns machine-readable validation, measurement, and geometry checks. When something breaks, the failure is fingerprinted, reduced to a minimal synthetic repro on your machine, and — only with your approval — filed upstream.

02 · Git-native

The model is text — a canonical, deterministic feature tree. Geometry (STEP, Gerbers, PDFs) is a build artifact, never source. So you get branching, merging, blame, pull requests, and semantic diffs — volume deltas, added features, before/after renders — for free. Releases are lockfile-pinned and reproducible forever. Stable entity identity is designed in from day one, so merges don't corrupt references.

03 · One substrate, two domains

Mechanical (b-rep solids, drawings) and electrical (schematics, PCB, Gerbers) are equal-weight domains on one part standard: every part publishes a domain-neutral interface — envelope, frames, typed ports — and assemblies mate against interfaces, never internals. The enclosure and the board live in one git history. Interface-semver makes compatibility machine-checkable.

Architecture: six seams

Everything swappable lives behind one of six narrow interfaces. The geometry kernel is proven technology (OpenCASCADE via OCP) — the value is the layers around it. A major architecture change means replacing one backend, not a rewrite.

What it looks like

# An agent (or you) builds a part through the same tools:
m = tools.model_new()
m = tools.feature_add(m, op="box", params={"dx":60, "dy":40, "dz":8})
m = tools.feature_add(m, op="cylinder", params={"radius":3.2, "height":8}, ...)
m = tools.feature_add(m, op="boolean", params={"kind":"cut"}, inputs=[...])

# ...then verifies instead of hoping:
tools.model_validate(m)   # watertight? self-intersecting? → structured report
tools.model_measure(m)    # volume, centroid — a deterministic oracle

# ...and ships real manufacturing outputs:
tools.model_export(m, fmt="step")     # → part.step
tools.model_drawing(m)                 # → dimensioned drawing (SVG/PDF)
tools.board_export_fab(b)              # → Gerber X2 + Excellon + pick-and-place

Status
